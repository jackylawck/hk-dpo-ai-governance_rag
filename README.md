# HK DPO AI Governance RAG Advisor (香港數字辦指引 RAG 智能顧問)

An advanced enterprise-grade GenAI regulatory consultation workstation powered by Retrieval-Augmented Generation (RAG). It performs semantic search and regulatory alignment strictly based on the HK DPO "Generative AI Technical and Application Guideline V1.1" (Dec 2025).
本系統為一套基於檢索增強生成 (RAG) 的企業級 AI 法規諮詢工作站，能針對香港數字政策辦公室《香港生成式人工智能技術及應用指引》V1.1 進行深度語義檢索與法規對齊。

---

### 🧱 Governance Methodology & Architecture (管治方法論與架構)

This project is powered by the **"Jacky Law Human-Centred AI Governance Stack"** and features Defense-in-Depth engineering controls:
本專案採用 **「Jacky Law 人本 AI 管治五層架構」** 進行頂層設計，並實作了多層防禦工程控制：

1. **ISO 42001 Cryptographic Audit Trail (密碼學審計軌跡):** All LLM reasoning and regulatory queries are automatically hashed (SHA-256) and recorded in a local, non-repudiable audit ledger (`.log`).
2. **Data Minimization (資料最小化):** Document embeddings are processed 100% locally using open-source `sentence-transformers` via HuggingFace. No proprietary corporate documents are transmitted to external embedding APIs.
3. **Failure-Safe Prompt Guardrails (防幻覺護欄):** The LLM is strictly constrained to the retrieved official context, proactively rejecting out-of-scope queries (e.g., traditional discriminative ML like Random Forest).

### 🛡️ Knowledge-Base Setup & Copyright Compliance (知識庫設定與版權合規)

**To strictly enforce copyright protection and information governance, this repository DOES NOT redistribute the official standard PDFs.**
**為嚴格遵守版權保護與資訊管治原則，本公開倉庫絕對不直接分發或公開任何官方指引文件原文。**

Users must prepare their own authorized copies of the guidelines and setup the local environment as follows:
使用者必須自行準備合法渠道獲取之指引文件，並依循以下步驟配置本地環境：

1. Clone this repository locally. (將本倉庫克隆至本地)
2. Create a folder named `knowledge_base/` in the root directory. (在根目錄建立 `knowledge_base/` 資料夾)
3. Download the official texts from the HK Digital Policy Office (DPO).
4. Place the downloaded PDFs into the `knowledge_base/` directory. (**Do not commit them to Git.**) (將下載的 PDF 放入該目錄，檔案已被 `.gitignore` 鎖定，絕對不會上傳)。
5. Add your `OPENAI_API_KEY` in a local `.env` file to trigger the LLM reasoning layer. (在本地配置 `.env` 檔案並填入 API 金鑰)。
